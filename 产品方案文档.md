FitMate AI — 产品方案文档


一、产品定位

一句话定位：面向健身入门者的 AI 私人教练 + 营养师，解决「不知道怎么练、不知道怎么吃、吃了记不住」三大痛点。

目标用户：有健身意愿但缺乏专业指导的 25-35 岁上班族，训练场地以居家为主。


二、用户痛点与解决方案

痛点1：训练盲目
具体场景：打开健身 App 不知道选哪个课程，跟着视频做也不知道对不对
产品方案：AI 根据个人画像（目标/水平/场地/时间）每日生成专属训练计划

痛点2：饮食失控
具体场景：知道要控制热量，但不会算也不知道吃多少
产品方案：AI 基于 BMR/TDEE 精准计算热量缺口，生成可执行的三餐方案

痛点3：记录低效
具体场景：拍照记、手写记、App 记，操作繁琐且记了没用
产品方案：拍照自动识别食物 + 热量，AI 对比今日预算给出即时反馈

痛点4：数据无感
具体场景：记录了体重腰围，但不知道变化趋势意味着什么
产品方案：AI 分析 7 天数据趋势，给出可读的健康洞察


三、功能架构

FitMate AI 包含六个功能模块，按优先级划分：

P0（每日核心诉求——用户打开 App 的理由）
  · 每日记录 —— 手动录入体重、腰围，数据本地持久化
  · 训练计划 —— AI 根据个人画像生成训练方案（动作 + 组数 + 次数）
  · 饮食计划 —— AI 基于热量目标生成三餐方案

P1（增强粘性的差异化功能）
  · 拍照识食 —— 拍照 → 视觉识别菜品 → AI 补救建议，形成闭环
  · 体脂分析 —— AI 分析近 7 天体重/腰围趋势，输出可读洞察

P2（基础可视化）
  · 历史数据 —— 体重/腰围变化折线图


四、核心流程设计

4.1 训练计划生成

用户画像（目标、水平、时间、场地）→ Prompt 模板（健身教练角色设定）→ DeepSeek LLM → 结构化训练方案（热身 + 正式训练 + 拉伸 + 注意事项）

交互设计：一键生成，结果即时展示，无需多步操作。每天可重新生成，保证计划不重复。

4.2 饮食计划生成

用户画像 → 热量计算引擎（BMR × 活动系数 ± 热量缺口）→ Prompt 模板（营养师角色设定）→ AI 三餐方案

设计要点：
  · 热量计算先于 AI 生成，保证目标可控——不用 AI 来算数学
  · 三餐热量分配：早餐 25% / 午餐 35% / 晚餐 30% / 加餐 10%
  · 用户可在侧栏调整饮食限制（如不吃猪肉），实时生效

4.3 拍照识食闭环（特色功能）

拍照上传 → 豆包视觉模型识别菜品 + 估算热量 → 今日热量预算对比（已吃 vs 剩余）→ AI 补救建议（晚餐调整等）

用户体验亮点：不局限于「识别这是什么食物」，而是形成「识别 → 计算 → 建议」的完整闭环，即时给出可执行的补救方案。


五、用户画像配置

产品将用户特征标准化为可配置参数，训练和饮食方案均基于这些参数个性化生成：

性别：男（影响 BMR 计算）
年龄：28（影响 BMR 计算）
身高：175 cm
体重：70 kg（每日更新）
目标：减脂（可选减脂 / 增肌）
目标体重：65 kg
训练水平：中级（可选新手 / 中级 / 高级）
训练时间：30 分钟（15-90 分钟可调）
训练场地：居家（可选居家 / 健身房）
饮食限制：无（自由填写，如素食、不吃猪肉）


六、技术选型决策

前端框架：Streamlit
选型理由：Python 全栈，快速原型验证，无需前后端分离

语言模型：DeepSeek
选型理由：性价比高，中文能力强，支持类 OpenAI SDK

视觉模型：豆包（火山方舟）
选型理由：国内可用，食物识别准确，API 调用简单

数据存储：CSV
选型理由：MVP 阶段无需数据库，单用户场景足够

部署平台：Streamlit Cloud
选型理由：免费，GitHub 联动，一键部署

核心原则：MVP 阶段追求开发效率最大化，每个选型都在 1 天内可跑通验证。


七、产品指标

核心功能可用率 —— 目标：6 个模块均可正常使用；衡量：功能验收 checklist
训练方案满意度 —— 目标：用户连续 3 天主动使用；衡量：自用观察
饮食热量准确度 —— 目标：AI 三餐总热量偏差低于 10%；衡量：手动核算对比
识食准确率 —— 目标：常见中餐菜品识别成功；衡量：10 个以上样本测试
每日使用耗时 —— 目标：3 分钟内完成全流程；衡量：实际计时


八、开发计划

阶段1：需求定义
任务：用户画像梳理 → 功能清单 → PRD 撰写
产出：本方案文档

阶段2：环境搭建
任务：API 接入跑通（DeepSeek + 豆包）
产出：可调用的 API 封装

阶段3：核心开发
任务：每日记录 + 训练计划 + 饮食计划
产出：MVP 可用

阶段4：增强开发
任务：体脂分析 + 拍照识食
产出：完整功能版本

阶段5：联调上线
任务：集成测试 + UI 优化 + 云端部署
产出：可部署版本

阶段6：自用验证
任务：连续使用 3 天 + Bug 修复
产出：稳定可用版本
