## 一、产品概述

### 1.1 产品名称
**FitMate AI**

### 1.2 产品定位
为自己量身定制的AI健身+饮食管理工具，解决“不知道怎么练、不知道怎么吃、吃了记不住”的日常问题。

### 1.3 核心价值

| 用户痛点 | 产品解决方案 |
|---------|-------------|
| 每天不知道练什么 | DeepSeek生成个性化训练计划（动作+组数+次数） |
| 不知道怎么吃 | DeepSeek生成减脂/增肌饮食方案（热量目标+三餐） |
| 吃错了不知道怎么办 | **拍照上传食物，豆包视觉模型直接识别**，AI生成补救建议 |
| 记录了数据但看不出意义 | DeepSeek分析体脂趋势，告诉你变化意味着什么 |

### 1.4 使用频率
每日1次，建议早上或睡前使用

### 3.2 P0功能详细说明

#### 功能F1：记录体重+围度

**交互流程**：
1. 页面显示今日日期
2. 输入框：体重(kg)、腰围(cm)
3. 点击【保存记录】按钮
4. 系统将数据追加到CSV文件
5. 提示“保存成功”

**数据格式**（CSV）：
```csv
日期,体重,腰围,备注
2026-01-15,70.2,72,
```

#### 功能F2：DeepSeek生成训练计划

**输入**：用户画像（性别/年龄/训练水平/目标/可用时间/场地）

**Prompt模板**：
```text
你是专业健身教练。根据以下信息生成今日训练计划：

【用户信息】
- 性别：{gender}
- 训练水平：{level}
- 目标：{goal}
- 可用时间：{time}分钟
- 场地：{location}

【输出要求】
严格按照以下Markdown格式输出：

## 今日训练计划
### 热身（5分钟）
- 动作1：xxx

### 正式训练（{time-10}分钟）
- 动作1：名称 × 组数 × 次数
- 动作2：名称 × 组数 × 次数
- 动作3：名称 × 组数 × 次数

### 拉伸（5分钟）
- 动作1：xxx

### 注意事项
- 一句话提醒
```

#### 功能F3：DeepSeek生成饮食计划

**热量计算公式**：
```python
# 基础代谢率（Mifflin-St Jeor公式）
if gender == "男":
    bmr = 10 * weight + 6.25 * height - 5 * age + 5
else:
    bmr = 10 * weight + 6.25 * height - 5 * age - 161

# 每日总消耗（活动系数1.375，轻活动）
tdee = bmr * 1.375

# 热量目标
if goal == "减脂":
    target_calories = tdee - 300  # 每天缺口300大卡
else:
    target_calories = tdee + 300  # 每天盈余300大卡
```

**Prompt模板**：
```text
你是专业营养师。根据以下信息生成今日饮食计划：

【用户信息】
- 目标：{goal}
- 热量目标：{target_calories} kcal
- 饮食限制：{restrictions}

【输出要求】
严格按照以下格式输出：

## 今日饮食计划（总热量：{target_calories} kcal）

### 早餐（约25%热量）
- 食物1：xxx
- 食物2：xxx

### 午餐（约35%热量）
- 食物1：xxx
- 食物2：xxx

### 晚餐（约30%热量）
- 食物1：xxx
- 食物2：xxx

### 加餐（可选，约10%热量）
- 建议：xxx

### 饮水提醒
- 建议饮水量：{weight * 30} ml
```


### 3.3 P1功能详细说明

#### 功能F4：DeepSeek体脂趋势分析

**输入**：最近7天的体重+腰围数据

**Prompt模板**：
```text
你是健身数据分析师。用户最近7天的数据如下：

体重(kg)：{weight_list}
腰围(cm)：{waist_list}

请分析：
1. 体脂变化趋势（下降/平台/上升）
2. 变化速度是否合理（每周减重0.5-1kg为健康）
3. 一句鼓励或提醒

输出格式：3行短文本
```

#### 功能F5：豆包视觉模型当餐分析（核心亮点）

**用户流程**：
1. 用户点击【上传图片】或【拍照】
2. 选择食物照片
3. 系统调用豆包视觉模型 → 识别菜品并估算热量
4. 调用DeepSeek生成补救建议
5. 展示结果

**界面布局**：
```
┌─────────────────────────────────────────┐
│  📸 记录今日吃的食物                      │
│  ┌─────┐                                │
│  │ 📷  │ 点击拍照/从相册选择              │
│  └─────┘                                │
│                                          │
│  已上传：午餐.jpg                         │
│  [识别食物]  ← 点击触发                   │
├─────────────────────────────────────────┤
│  🍽️ 识别结果：                           │
│  ┌─────────────────────────────────────┐│
│  │ 宫保鸡丁    158 kcal  置信度92%      ││
│  │ 米饭        116 kcal  置信度88%      ││
│  └─────────────────────────────────────┘│
│  合计：274 kcal                          │
│  今日剩余预算：126 kcal                   │
│                                          │
│  💡 补救建议：                            │
│  晚餐建议减半主食，增加20分钟快走         │
└─────────────────────────────────────────┘
```

**豆包视觉模型调用方法**：
```python
from openai import OpenAI

client = OpenAI(
    base_url="https://ark.cn-beijing.volces.com/api/v3",
    api_key="ark-e94ef6be-23b0-43c4-8dfc-d3ce8282daec-e1177",
)

response = client.responses.create(
    model="ep-20260520040238-bvzw",
    input=[
        {
            "role": "user",
            "content": [
                {
                    "type": "input_image",
                    "image_url": image_base64_or_url
                },
                {
                    "type": "input_text",
                    "text": "请分析这张图片中的食物，列出菜品名称和预估热量（单位：千卡），以JSON格式返回。格式：{'dishes': [{'name': '菜品名', 'calories': 数字}], 'total_calories': 数字}"
                },
            ],
        }
    ]
)
```

**DeepSeek补救建议Prompt**：
```text
用户今日吃了以下食物：
{豆包返回的菜品名称和热量列表}

用户今日剩余热量预算：{remaining_calories} kcal

请输出（严格3行）：
1. 热量评估：xxx
2. 预算对比：超出/剩余 xxx kcal
3. 补救建议：一句话建议（如：晚餐减半主食、增加运动等）
```
